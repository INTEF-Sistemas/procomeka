# Glosario

Términos usados en el proyecto con su definición canónica. Cuando un término aparezca en código, documentación o conversación, debe usarse en el sentido aquí definido.

---

## Recurso educativo

Cualquier material digital (o digitalizable) que pueda ser usado en un contexto de enseñanza o aprendizaje. Incluye documentos, presentaciones, vídeos, actividades, aplicaciones interactivas, secuencias didácticas, etc.

## Recurso educativo abierto (REA)

Recurso educativo publicado con licencia abierta (Creative Commons u equivalente) que permite su reutilización, adaptación y redistribución.

## Metadato

Campo descriptivo asociado a un recurso: título, descripción, autor/a, nivel educativo, materia, tipo, licencia, idioma, etc.

## Perfil de aplicación

Conjunto de campos de metadatos seleccionados y adaptados para un uso concreto, con sus reglas de obligatoriedad y validación.

## Vocabulario controlado

Lista cerrada de valores válidos para un campo de metadatos. Ejemplo: niveles educativos, tipos de recurso, idiomas ISO 639-1.

## Taxonomía

Vocabulario controlado con estructura jerárquica. Ejemplo: área de conocimiento → materia → submaterias.

## Curación

Proceso editorial de revisión, enriquecimiento y validación de los metadatos de un recurso para garantizar su calidad y utilidad.

## Estado editorial

Fase del ciclo de vida de un recurso: borrador, en revisión, validado, destacado, archivado, rechazado.

## Colección

Conjunto de recursos agrupados con un criterio curado: temática, nivel, autor/a, evento, etc.

## Itinerario

Secuencia ordenada de recursos pensada para una progresión de aprendizaje.

## Ingestión

Proceso de importación de recursos desde una fuente externa al catálogo.

## Cosecha (harvest)

Importación automática de metadatos desde otro repositorio usando un protocolo estándar (típicamente OAI-PMH).

## ADR

Architecture Decision Record. Documento que registra una decisión de arquitectura o tecnología: qué se decidió, por qué, qué alternativas se consideraron y qué consecuencias tiene.

## Épica

Iniciativa de producto amplia que agrupa varias features con un objetivo funcional o estratégico común.

## Feature

Entrega concreta dentro de una épica, con criterios de aceptación definidos.

## LOM

Learning Object Metadata (IEEE 1484.12). Estándar de metadatos para objetos de aprendizaje, referencia principal del sector educativo.

## Dublin Core

Conjunto mínimo de metadatos descriptivos ampliamente adoptado para interoperabilidad entre repositorios.

## OAI-PMH

Open Archives Initiative Protocol for Metadata Harvesting. Protocolo estándar para cosecha de metadatos entre repositorios digitales.

## LRMI

Learning Resource Metadata Initiative. Extensión de Schema.org para describir recursos educativos en la web.

## LTI

Learning Tools Interoperability. Estándar IMS para integrar herramientas externas en plataformas LMS (Moodle, Canvas, etc.).

## Procomún (legacy)

El sistema actual de repositorio de recursos educativos del Ministerio de Educación de España, al que este proyecto sirve como sustituto moderno.
